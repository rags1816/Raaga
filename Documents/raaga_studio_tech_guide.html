Audio Context Init

function initAC(){
  AC = new(window.AudioContext||
           window.webkitAudioContext)();
  masterGain = AC.createGain();
  analyserNode = AC.createAnalyser();
  analyserNode.fftSize = 512;
  masterGain.connect(analyserNode);
  analyserNode.connect(AC.destination);
  reverbNode = AC.createConvolver();
  reverbNode.buffer = makeImpulseResponse(AC,1.2,2.0);
  // ...reverb routing...
}
Key: AudioContext created only on user gesture (startApp click) to satisfy browser autoplay policy. Checks AC.state==='suspended' and calls AC.resume() before every note.

Instrument Synthesis Pattern

// Each synth follows this pattern:
function synthHarmonium(freq){
  const t = AC.currentTime, dur = 0.7;
  const g = AC.createGain();
  // Attack + Decay envelope
  g.gain.setValueAtTime(0, t);
  g.gain.linearRampToValueAtTime(0.22, t+0.015);
  g.gain.exponentialRampToValueAtTime(0.001, t+dur);
  // Oscillators (sawtooth with detuning)
  [0,4,-3].forEach(detune => {
    const o = AC.createOscillator();
    o.type = 'sawtooth';
    o.frequency.value = freq * pow(2, detune/1200);
    o.connect(bp); // bandpass filter
    o.start(t); o.stop(t+dur+0.05);
  });
  connect(g); // routes to dry+reverb
}

Known Constraints & Notes

  • File size: 159KB — too large for Claude's artifact inline viewer (~100KB limit). Must be downloaded and opened locally in a browser.
  • Audio synthesis: All instruments are synthesised in real-time using Web Audio API oscillators. No sample files or external audio CDN.
  • Tone.js: Completely removed. The old codebase used Tone.js which caused silent failures due to CDN loading issues. Current build uses native Web Audio API only.
  • totalXP: Must be declared exactly once with let. A second let totalXP declaration anywhere in the same script scope causes a fatal SyntaxError in strict mode, preventing the entire app from loading.
  • Taar Saptak: Playable by mouse/touch only — no keyboard hotkeys assigned to upper octave (would conflict with browser shortcuts). Rendered at same physical size as other octaves via equal CSS dimensions, distinguished by slight opacity:.85.
  • setInterval taal: Rhythm uses setInterval (not Tone.Transport or Web Audio scheduler). Drift at very high BPM is acceptable for practice purposes.
  • localStorage: Stores raagaStudioXP as a string integer. No other data is persisted.
  • Touch support: All interactive elements have touch-action:manipulation to eliminate 300ms tap delay. touchstart with passive:false + preventDefault() used for keys and pads to prevent scroll interference.
