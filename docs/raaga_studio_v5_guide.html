⚙️ Audio Engine — How It Works

Sample Playback Flow

  • Key pressed → playNote() called
  • Checks if instrument is in SAMPLE_INSTRUMENTS set
  • Calls getSampleBuffer() → fetches WAV from sounds/ folder
  • Result cached in soundCache{} map
  • playSample() fires AudioBufferSourceNode
  • If fetch fails → synthFallback() called

Octave Resolution Logic

  • octOff=-1 (Mandra) → uses octave 1 sample
  • octOff=0 (Madhya) → uses octave 1 sample
  • octOff=1 (Taar) → uses octave 2 sample
  • If octave 2 missing → plays oct 1 at 2× speed
  • If octave 1 missing → plays oct 2 at 0.5× speed
  • If both missing → synth fallback fires
// Sample path construction (in getSampleBuffer): sounds/{instrument}/{instrument}_{octave}_{noteNum}_{swara}_{note}.wav // Example for Sitar, Pa (semitone 7), Madhya Saptak: → SEMI_TO_SWARA[7] = 'P' → SAMPLE_NOTES[7] = 'G' → noteNum = '08' → path: sounds/sitar/sitar_1_08_P_G.wav
💡 Cache behaviour: First play of each note fetches from disk. Subsequent plays are instant (memory buffer). The cache persists for the entire session — no re-fetching.

🔧 Troubleshooting

ProblemCauseFix
Sounds are synthesised, not realsounds/ folder not found, or files missingCheck folder is next to the HTML. Open DevTools → Network tab — look for 404 errors on .wav files
First note is slow/delayedWAV loading from disk on first playNormal. After first play each note is instant. Preload runs on startup for Sa, Pa, S'.
No sound at allBrowser blocked autoplayClick "Start Playing" first. Ensure user interaction before audio plays.
Wrong octave soundsOnly one octave folder providedApp auto pitch-shifts. Provide both octave 1 and 2 folders for best quality.
Percussion sounds odd on keyboardPercussion samples are non-melodic hitsBest used for rhythm/backing. Tabla etc. in melody mode uses synth fallback.
File not found in browserOpened HTML directly (file:// protocol)Serve from a local server: python3 -m http.server 8000 then open localhost:8000
⚠ CORS / file:// protocol: If you double-click the HTML to open it in a browser (file:// URL), fetch() calls to load WAV files may be blocked by the browser's security policy. The synth fallback will work, but real samples won't load. To use real samples, either: (a) use Chrome with --allow-file-access-from-files, or (b) run a local web server.

Quick Local Server Setup

# In your Music/ folder, open Terminal and run: python3 -m http.server 8000 # Then open in browser: http://localhost:8000/raaga_studio_v5.html

➕ Adding More Instruments in Future

To add a new real-sample instrument (e.g. "esraj"):

  1. Create folder: sounds/esraj/
  2. Add 12–24 WAV files following the naming convention
  3. In the HTML, add 'esraj' to the SAMPLE_INSTRUMENTS Set
  4. Add a <option value="esraj"> to the instrument dropdowns
  5. Add a synth fallback case in synthFallback() (e.g. map to synthSarod)
